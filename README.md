# Pasos a seguir para la realización del proyecto

1. Se creó un repositorio en GitHub para almacenar y documentar todo el código del proyecto y para tener un control más riguroso de los archivos del mismo, también se incluyeron como colaboradores a todos los que participaron en el.

2. Se desarrolló la sección superior del sitio la cual consta de una barra de navegación, con los distintos enlaces del sitio, y un banner que es una imagen en la cabecera del sitio, además se desarrollaron las demás paginas cuyos enlaces están en la barra de navegación, ya que cada enlace del sitio dirige al usuario a otra página del mismo. Finalmente, todo el código que se generó en este paso del proyecto se cargó en una nueva rama llamada iss1 dentro el repositorio remoto de GitHub.

3. Se desarrolló la sección de servicios, la cual consta de un contenedor que a su vez contiene tres contenedores, cada uno de ellos describe uno de los servicios que ofrece la empresa dueña del sitio web, estos contenedores también contienen un enlace a una nueva página donde se describiría con más detalle cada uno de los servicios antes mencionados. El código necesario para crear esta sección del proyecto se almacenó en el repositorio de GitHub bajo la rama iss2.

4. Se desarrolló la sección de noticias, la cual consta de un contenedor que a su vez contiene cuatro contenedores más pequeños ordenados como una tabla, en filas y columnas, dos filas y dos columnas, en cada uno de estos contenedores se muestra el título de una noticia junto con una descripción resumida de esta y un enlace para leer la noticia completa, cada uno de estos enlaces redirige al usuario a una página nueva que contiene la noticia completa, este aporte se almacenó en el proyecto de GitHub en una nueva rama llamada iss3 del proyecto creada para tal fin.

5. Se desarrollo la sección donde se muestra la información de los miembros del equipo responsables del proyecto, esta sección contiene a su vez cinco contenedores más pequeños ordenados en fila, cada uno de estos contenedores muestra la información de uno de los miembros del grupo de trabajo responsable del proyecto, el código implementado para la realización de esta sección del proyecto se cargó bajo la rama iss4.

6. Se finalizó el desarrollo de la página con la implementación de un pie de página, este se encuentra al final de la página principal y muestra algunos datos importantes del proyecto, este footer o pie de página se dividió en dos partes, una de ellas contiene información del sitio web y la otra contiene información de contacto de los miembros del equipo que realizó el proyecto, el código escrito para la implementación de este aporte se almacenó en el repositorio de GitHub bajo la rama de trabajo iss5.

7. Se unificaron todas las ramas desarrolladas con anterioridad en una única rama llamada dev, una vez allí, se verificó que todo el código funcionara correctamente en su conjunto.

8. Se unió la rama dev con la rama principal del proyecto (main) con esto se dio por finalizado el proyecto.
